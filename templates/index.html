This is webpage, where you can test your strategies. Apart from web interface this page contains api. Therefore programming your strategies can be easier.

Here are some examples of endpoints, that might be useful to you:

  • '/api/game/new/' - get request takes argument player1, player2, player3 (optional) and player4 (optional) values are names of player returns session id
  • /api/cardlist/ - api version of cards that are used in games (should not change in time)
  • /game/api/session/<session_id> - here is info about actual game status this is data you get for your strategy
  • /game/api/<session_id> - here you send your move format of your data is based on what you want to do
    • {'action': 'pass'} if you want to skip your turn
    • {'action': 'hold', 'card_id': <card_id>} if you want to hold the card
    • {'action': 'buy', 'card_id': <card_id>} if you want to buy the nonupgrade card
    • {'action': 'buy', 'card_id': <card_id> 'upgrade_from': <card_id>} if you want to buy the upgrade card
